pytest 框架

发表于 2023-08-12 更新于 2023-08-14 分类于 技术 Valine：
本文字数： 7.4k 阅读时长 ≈ 7 分钟

pytest是一个单测框架；能对定义的测试对象进行测试，并且可以针对测试对象进行前后置处理，对测试结果进行断言，对执行过程中的异常进行干预，并且可以收集测试结果以生成xml格式的测试报告

大纲

原理

pytest是一套单测框架；有着自己的巡检方法，他会自动查找整个包中以test开头/结尾的模块，并查找其中test开头/结尾的方法/类（类很特殊必须要以Test开头，且类中不能有__init__方法），全部保存在内存中，在触发执行时，会根据调度指令来确定自动执行的用例范围，来进行执行； 同时执行用例时做了装饰器操作，将用例作为参数传递到pytest框架中执行。

使用场景

  • __用例自动单测场景：__如接口自动化、UI自动化、各类用例场景；
  • __通用前后置处理场景：__如数据库操作前的建立连接，操作完成后的断开连接；
  • __参数化操作场景：__如多设备操作、各种UI操作、各种接口操作；

如何使用

分为：调度/执行、前后置处理（固件）、断言操作、异常处理、参数化、定制报告

调度/执行

调度有两种方式：命令行、pytest.main()
pytest.main()实际也是调用命令行来进行的操作，我们程序一般使用pytest.main()来进行调度；注意pytest.main([-m 标记]);第一个-m是选项，第二个是参数;pytest.main()会将当前模块加入缓存，因此不建议执行过程中调用多次，最好只调用一次。

定制报告

  • 前后置处理
    固件的作用是提供了一个固定的基线，能重复的运行；固件装饰器的作用是注册一个固件，
    测试用例使用固件的方式，调用的是固件的形参没，实际生效的是固件的返回对象作为入参
    setup、downup\fixture(必须放置在conftest模块中)，其中预制方法的yield前属于前置，yield后属于后置操作(这个后置操作，只能对前置产生的实例对象生效)；
    除了yield的后置操作，支持上下文的对象，还可以使用with本身提供的隐式的清理操作；
    除了这种方式来实现后置还可以使用，request.addfinalizer(后置方法)来实现后置，只是这个方法一定要放置预处理的固件的后面
    @pytest.fixture(scope=’function/class/module/session’,autouse=True),scope是用来声明作用域的。前置后置可以有多个，可以显示调用，也可以不调用通过默认的自动调用来隐式调用(autouse=True，默认为true)；根据作用域大小和conftest中的先后顺序会有调用顺序，同时固件还可以彼此调用（作为参数）后置的顺序正好相反；前后置方法除了方法可以用，类也可以用@pytest.mark.usefixtures(‘class_scope’,’第二个固件’)；这个除了类可以使用外，方法也可以使用
    除了显示调用，隐式自动调用，还可以设定按固定条件调用,估计还可以指定别名，指定别名后，原来的名字不可用；
    参数化，不单可以针对测试函数（），固件也可以参数化
    固件放在conftest中是为了方便复用和注册
    固件fixture可用于共享数据
    conftest作用域和这个文件的位置也有很多关系，如果需要对当前包的整体生效，那么conftest文件需要放置在包的根目录下
    可以在固件中对测试用例打标记，并给测试用例分配测试id
    @pytest.fixture(params=[(‘3+5’, 8),pytest.param((‘6*9’, 42),
                                  marks=pytest.mark.xfail,
                           id='failed')])
    和用例参数化等价
    @pytest.mark.parametrize(‘test_input, expected’,[(‘3+5’, 8),
    pytest.param(‘6*9’, 42, marks=pytest.mark.xfail, id=’failed’)]) //注意这是两组参数
    固件除了在conftest中定义外，还可以在用例模块本地来复写
  • 作用域
    conftest.py（这是pytest的插件，pytest会自动调度）：放置在当前目录下，此目录及旗下所有子目录都生效，会执行以test开头/结尾的模块中的以test开头或结尾的方法；
    使能autouse的fixture，先于其同级别的其它fixture实例化；同时autouse的fixture测试用例显示声明（形参），不声明的不会被实例化；
    固件本身除了可以调用其他固件外；还可以给固件传参，从而使得固件可以实现通用化；
    固件除了返回实例对象或参数给测试用例调用外，固件还可以直接返回工厂函数
    可以对固件进行参数化。但是固件参数化后会创建多个实例，因此单个测试用例也会执行多次
    @pytest.fixture(scope=’module’, params=[‘smtp.163.com’, “mail.python.org”])
    def smtp_connection_params(request):
    server = request.param

如果不想执行多次，可以使用模糊匹配来指定执行实例如pytest -k 163 用例
低级别的固件可以调用高级别作用域的固件，反之则不行
固件参数化时，应该是一个参数实例使用完后，才会使用第二个
可以有多个conftest.py文件，生效的顺序是就近原则，优先当前模块-当前文件夹-上一级文件夹中的
复写操作，除了在conftest中，用例模块中，还可以在用例的参数化中进行复写
@pytest.mark.parametrize(‘已定义固件名’, [‘directly-overridden-username’])
参数化的fixture和非参数化的fixture同样可以相互覆盖；同名覆盖，按距离优先原则

  • 生效机制
    auto-test，test开头/结尾模块中的test开头/结尾的方法，类也满足此条件；（但类必须要以Test开头，且类中不能有__init__方法）
  • 如何调用
    pytest test_.py
    pytest.main()
    可以指定执行特定的函数使用（::/-k/-m+mark装饰器标记）；也可以反向操作，指定要跳过的函数（pytest.mark.skipif）
    可以对函数执行失败进行定义，显示的提示失败使用装饰器（pytest.mark.xfail）
    参数化装饰器：pytest.mark.parametrize(argnames, argvalues)
    可设置最大允许的失败数，如果设置了达到最大时，这pytest会中断执行，否则无最大次数限制；（pytest –maxfail==2）
    可通过命令行执行特定包中的用例/特定目录下/特定模块/特定方法/特定方法的特定参数化参数
    pytest –pyargs pkg.testing
    pytest testing/pytest test_mod.py/pytest test_mod.py::test_fun
    $ pytest -q -s test_nodeid.py::TestNodeId::test_two[1-1]
    模糊匹配：执行含特定字符或不含特定字符的模块/类/函数：
    pytest -k “_class and not two”
    还可以为用例添加标记。从而指定特定标记的用例
    @pytest.mark.标记名
    def test_one()
    注意pytest.main([-m 标记])执行方法和命令行执行是相同的;第一个是选项，第二个是参数;注意main（）会将模块加入缓存，因此不建议执行过程中调用多次，最好只调用一次
    命令行执行的各选项
  • 断言
    def test_
    assert
    断言除了常规的判定式；还有指定异常的检测
    def test_mytest():
    with pytest.raises(SystemExit) as excinfo:
      f()        //raise SystemExit(1)抛出了指定异常
    assert ‘456’ in str(excinfo.value) //excinfo是上下文的实例，且在上下文环境中raises必须是最后一行,因为，f()同层级的后续行不会被执行
    除了断言还可以设置断点：使用pdb.set_trace()，只影响断点用例
    可以给异常加标记：出现IndexError，则标记用例失败类型为xfail，这种用法一般用来标记未修复的bug
    @pytest.mark.xfail(raises=IndexError)
    def test_f():
    f()

不同数据类型的断言，不同地方都会标记出来
可以给断言失败，添加说明，可以用运算重载符，出现系统提示的字符串异常时，重载为我们想要的内容；第二种是利用钩子pytest_assertrepr_compare，来定义异常
断言的异常信息，会存储在本地，如需禁止可以在conftest.py文件中加入这些配置实现：import sys
sys.dont_write_bytecode = True
关闭断言的失败提示功能：使用再需要去掉的模块中添加模块的docstring中添加PYTEST_DONT_REWRITE字符串；调度用例时添加 pytest –assert=plain选项

  • 固定固件
    pytest有现成的一些估计，可以直接在测试单元中调用，或者在自定义的固件中调用
    查看现成的固件的方法：pytest -q -v –fixtures
  • 结果
    为测试报告中的测试用例添加额外的信息，使用record_property fixture
    def test_record_property(record_property):
    record_property(“test_id额外信息”, 10010额外信息值)
    assert 1
    也可以在测试报告中加入用例的标记信息 //修改pytest_collection_modifyitems钩子方法，添加对test_id标记的支持：
    可以为测试报告添加用例额外的属性信息 //使用record_xml_attribute fixture
    可以在测试报告中为整个测试集添加额外的信息

可以在报告中为用例创建链接
pytest –pastebin=all //为所有用例创建链接
pytest –pastebin=failed //为失败用例创建链接
